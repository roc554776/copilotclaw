```

GET  /api/logs                             → 200 { logs: string[] } (recent log lines from ring buffer)
GET  /api/sessions/:sessionId/messages     → 200 unknown[] | 404 (copilot session conversation history; proxied from agent via IPC)
GET  /api/channels                         → 200 Channel[] (?includeArchived=true to include archived)
POST /api/channels                         → 201 Channel
PATCH /api/channels/:id                    → 200 Channel (archive/unarchive via { archived: boolean }, model setting via { model: string|null })
GET  /api/channels/:id/draft               → 200 { draft: string }
PUT  /api/channels/:id/draft               → 200 { ok: true } (save draft text for a channel)
GET  /api/channels/pending                 → 200 { [channelId]: count }
GET  /api/channels/:channelId/messages              → 200 Message[] (?limit=N&before=rowid, reverse-chronological, cursor-based pagination via rowid)
GET  /api/channels/:channelId/intents/:agentId      → 200 IntentEntry[] (?limit=N, newest first; returns intents from SQLite intents table — v0.79.0)
POST /api/channels/:channelId/messages              → 201 Message | 400 (requires sender field: "user"|"agent"|"cron"|"system"; returns 400 if sender missing or invalid; user/cron/system messages go to pending queue)
POST /api/channels/:channelId/messages/pending      → 200 Message[] | 204 (drain all pending user messages)
GET  /api/channels/:channelId/messages/pending/peek → 200 Message | 204 (oldest pending, non-destructive)
POST /api/channels/:channelId/messages/pending/flush → 200 { flushed: count }
GET  /status                               → 200 HTML standalone SystemStatus page (same data as modal)
GET  /sessions/:sessionId/events          → 200 HTML session events stream page (flat/nested toggle, auto-scroll)
POST /api/session-events                  → 201 { ok: true } (agent posts SDK session events)
GET  /api/sessions/:sessionId/events      → 200 SessionEvent[] (events for a session; accepts ?limit=N&before=id&after=id for cursor-based pagination via getEventsPaginated)
GET  /api/sessions/:id/events/stream      → SSE stream; session-scoped SSE endpoint always available (v0.74.0); client subscribes via addSessionClient; event push only occurs when sessionEventStore is non-null (setOnAppend wires broadcastToSession in daemon.ts); no sessionEventStore → stream stays open but silent
GET  /api/session-events/sessions         → 200 string[] (session IDs with events)
GET  /api/token-usage                     → 200 TokenUsageEntry[] (?hours=5 default, or ?from=ISO&to=ISO for custom range; proxied from SessionEventStore.getTokenUsage)
GET  /api/token-usage/timeseries          → 200 TimeseriesPoint[] (?hours or ?from=ISO&to=ISO, ?points=N, ?movingAverageWindow=sec; time-bucketed per-model usage with consumption index and optional moving average)
GET  /api/cron                             → 200 CronJobStatus[] (cron job statuses)
POST /api/cron/reload                      → 200 { ok: true } (reload cron scheduler from config)
PUT  /api/cron                             → 200 { ok: true } (save cron jobs to config and reload scheduler)
POST /api/sessions/:id/stop                → 200 { ok: true } (stop a physical session; requires agentManager + sessionOrchestrator)
POST /api/sessions/:id/end-turn-run        → 200 { ok: true } (end turn run — idleSession instead of suspendSession; requires agentManager + sessionOrchestrator)
GET  /api/system-prompts/original         → 200 SystemPromptSnapshot[] (all captured original prompts)
POST /api/system-prompts/original         → 201 { ok: true } (agent posts captured original system prompt)
GET  /api/system-prompts/original/:model  → 200 SystemPromptSnapshot | 404
POST /api/system-prompts/effective        → 201 { ok: true } (agent posts effective system prompt)
GET  /api/system-prompts/effective/:id    → 200 { sessionId, model, prompt, capturedAt } | 404
GET  /                                    → 200 HTML dashboard (status bar + channel tabs + chat UI)
```

### IPC Protocol (newline-delimited JSON over Unix domain socket)

```
→ {"method":"status"}
← {"version":"0.17.0","bootId":"uuid","startedAt":"...","sessions":{"sess-id":{"status":"waiting|suspended","startedAt":"...","boundChannelId":"ch-id","copilotSessionId":"...","physicalSession":{...},"subagentSessions":[...]}}}

→ {"method":"session_status","params":{"sessionId":"sess-id"}}
← {"status":"waiting|processing|suspended|stopped","startedAt":"...","processingStartedAt":"...","boundChannelId":"ch-id"}
  (or {"status":"not_running"} if no session exists)

→ {"method":"stop"}
← {"ok":true}  → graceful shutdown of all sessions

→ {"method":"quota"}
← { ...quota object } | {"error":"no active session"}

→ {"method":"models"}
← { ...models object } | {"error":"no active session"}

→ {"method":"session_messages","params":{"sessionId":"copilot-sess-id"}}
← [ ...message objects ] | {"error":"session not found"} | {"error":"missing sessionId"}

→ {"method":"stream"}
← {"ok":true}  → connection upgrades to persistent bidirectional stream (newline-delimited JSON)
```

**AbstractSessionStatus values (v0.18.0, expanded v0.58.0)**:
- "new" — initial status on startSession creation (before physical session starts)
- "starting" — session initializing, copilotClient not yet bound
- "waiting" — idle, awaiting user input (keepalive tool polling gateway)
- "notified" — message arrived on a waiting session (set by cron scheduler and server.ts)
- "processing" — handling tool calls or LLM requests
- "idle" — physical session ended normally via idleSession(); abstract session preserved but not active; excluded from hasActiveSessionForChannel, idleAllActive, reconcileWithAgent
- "suspended" — physical session ended unexpectedly or max age reached, abstract session preserved for later revival (physicalSessionId retained); persisted to disk if persistPath configured
- "stopped" — session fully removed via explicit stopSession()

```

copilotclaw_send_message(message)   — dynamically registered from toolDefinitions config; dispatches via gateway RPC (tool_call)
copilotclaw_list_messages(limit?)   — dynamically registered from toolDefinitions config; dispatches via gateway RPC (tool_call)
copilotclaw_intent(intent)          — dynamically registered from toolDefinitions config (v0.70.0); dispatches via gateway RPC (tool_call); gateway records intent to intentsStore; must not be called standalone (constraint in both CHANNEL_OPERATOR_PROMPT and WORKER_PROMPT)
```

Tool registration:
- **copilotclaw_wait**: always built-in (hardcoded in createChannelTools)
- **Other tools**: dynamically registered from AgentPromptConfig.toolDefinitions (default: copilotclaw_send_message, copilotclaw_list_messages, and copilotclaw_intent); all dispatch via gateway RPC (tool_call request-response); gateway offline → graceful error (no throw)

Tool availability:
- **channel-operator** (parent): receives all tools (wait + dynamically registered tools including copilotclaw_intent)
- **worker** (subagent): receives dynamically registered tools except wait (copilotclaw_list_messages, copilotclaw_send_message, copilotclaw_intent); never receives copilotclaw_wait, preventing deadlock if worker attempts to block

### SDK Hooks

- All SDK hooks registered via generic `makeHookHandler(hookName)` that sends an RPC (type "hook") to gateway with the hook name and event data, receives response with additionalContext; enables gateway-side hook logic without agent code changes per hook
- `postToolUseFallback` method: fallback behavior when gateway is offline (RPC fails); performs pending message peek and injects notification into additionalContext
- `onPostToolUse` — registered via makeHookHandler; gateway-side logic (daemon.ts onHook) gates on `copilotclaw_wait` (the sole parent-exclusive tool) to avoid injecting reminders/notifications into subagent tool calls (SDK hook system provides no mechanism to distinguish parent vs subagent calls); performs two checks:
  - Pending message detection: peeks channel for pending user messages, injects notification into additionalContext
  - System prompt reinforcement (v0.15.0+): fires periodic reminders via `<system>` tagged additionalContext when context usage crosses 10% increments or after compaction; gateway daemon maintains reminderStates Map per session (needsReminder, lastReminderPercent, currentUsagePercent) — updated by onSessionEvent handler for session.usage_info and session.compaction_complete events; systemReminder from AgentPromptConfig injected by gateway onHook handler
