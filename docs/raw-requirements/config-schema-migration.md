# Config Schema Migration (raw requirement)

<!-- 2026-03-27 -->

## 設定ファイルの互換性

- バージョン更新で、設定ファイルの互換性が壊れないようにする
- 設定ファイルにスキーマバージョンを追加する
- バージョンが古い設定ファイルを読み込んだとき、マイグレーションを自動で行う
  - 段階的にマイグレーションを行う必要がある
  - つまり、v1→v2, v2→v3, v3→v4 のようなマイグレーション関数を全て用意しておいて、それらを順番に適用することで、古いバージョンの設定ファイルを最新のバージョンにマイグレーションできるようにする
- 設定ファイルのスキーマバージョンと、アプリのバージョンは揃える必要はない（揃えてもよい）
